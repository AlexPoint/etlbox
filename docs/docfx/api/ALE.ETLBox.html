Classes

AbortLoadProcessTask

Will set the table entry for current load process to aborted.

AddFileGroupTask

This task will add a filegroup to the database.

AdomdConnectionManager

BlockTransformation<TInput>

A block transformation will wait for all data to be loaded into the buffer before the transformation is applied. After all data is in the buffer, the transformation is execution and the result posted into the targets.

CalculateDatabaseHashTask

Calculates a hash value of the database. It will use only the schemas given in the property SchemaName for the calculation. The hash calcualtion is based only on the user tables in the schema.

CleanUpLogTask

Removes log data older than the specified days to keep.

CleanUpSchemaTask

Tries to remove all database objects from the given schema(s).

ConnectionString

ControlFlow

Contains static information which affects all ETLBox tasks. Here you can set default connections string, disbale the logging for all processes or set the current stage used in your logging configuration.

CreateDatabaseTask

Will create a database if the database doesn't exists.

CreateIndexTask

Creates an index if the index doesn't exists.

CreateLogTablesTask

Will create two tables: etl.Log and etl.LoadProcess. Also it will create some procedure for starting, stopping and aborting load processes. If logging is configured via a NLog config, these tables contain log information from the tasks.

CreateSchemaTask

Creates a schema if the schema doesn't exists.

CreateTableTask

Creates a table. If the tables exists, this task won't change the table.

CRUDProcedureTask

Creates or updates a procedure.

CRUDViewTask

Creates or updates a view.

CSVSource

Reads data from a csv source. While reading the data from the file, data is also asnychronously posted into the targets.

CustomDestination<TInput>

Define your own destination block.

CustomSource<TOutput>

Define your own source block.

CustomTask

A custom task allows you to run your own code (defined as an Action object), with additionally logging in place. (TaskType: CUSTOM)

DataTypeConverter

DbConnectionManager<Connection, Command>

DBDestination<TInput>

A database destination defines a table where data from the flow is inserted. Inserts are done in batches (using Bulk insert).

DBSource<TOutput>

A database source defines either a table or sql query that returns data from a database. While reading the result set or the table, data is asnychronously posted into the targets.
